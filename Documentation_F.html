1️⃣ Nettoyage des textes

Le texte est nettoyé de manière multilingue :

  • Détection automatique de la langue.
  • Conversion en minuscules.
  • Suppression de la ponctuation.
  • Suppression des stopwords selon la langue détectée (anglais par défaut).
def clean_text(text):
    if pd.isna(text):
        return ""
    try:
        lang = detect(text)
    except:
        lang = "en"
    stop_words = set(stopwords.words(lang)) if lang in stopwords.fileids() else set(stopwords.words("english"))
    text = text.lower()
    text = text.translate(str.maketrans("", "", string.punctuation))
    tokens = [word for word in text.split() if word not in stop_words]
    return " ".join(tokens)

2️⃣ Analyse de sentiment avec VADER

Après nettoyage, une analyse de sentiment est effectuée :

  • VADER (Valence Aware Dictionary and sEntiment Reasoner) est un outil d’analyse de sentiment basé sur un lexique et des règles, spécifiquement conçu pour détecter les sentiments exprimés sur les réseaux sociaux, et qui fonctionne également bien sur des textes provenant d’autres domaines. Le lexique VADER a été validé empiriquement par plusieurs juges humains indépendants. Il intègre un lexique de sentiment « gold standard » particulièrement adapté aux contextes de type microblog.
  • Le score "compound" est utilisé pour classer le texte en positive, neutral ou negative.
  • Les colonnes review_title et review_body sont combinées pour former full_review avant analyse.
def get_sentiment(text):
    if pd.isna(text) or text.strip() == "":
        return "neutral"
    scores = sia.polarity_scores(text)
    compound = scores["compound"]
    if compound >= 0.05:
        return "positive"
    elif compound <= -0.05:
        return "negative"
    else:
        return "neutral"

df["full_review"] = df["review_title"].astype(str) + " " + df["review_body"].astype(str)
df["review_sentiment"] = df["full_review"].apply(get_sentiment)

3️⃣ Combinaison multi-modale pour prédiction

Après nettoyage et analyse de sentiment, les caractéristiques sont combinées :

  • Texte nettoyé / TF-IDF (titre, review)
  • ASIN : One-hot encoding
  • Prix : Normalisation
  • Image : ResNet18 features
  • Sentiment : peut être utilisé comme feature additionnelle
X_final = [TF-IDF texte nettoyé | One-hot ASIN | Prix normalisé | Features image | Sentiment]

5️⃣ Avantages

  • Multi-modale : texte, catégorie, prix, image et sentiment.
  • Nettoyage multilingue standardise le texte.
  • Analyse de sentiment capture l’opinion exprimée dans la review.
  • TF-IDF + ASIN + Prix + Image + Sentiment : combinaison complète pour prédiction précise.
  • CatBoost gère efficacement les données hétérogènes.
